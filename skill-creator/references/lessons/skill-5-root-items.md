The canonical root layout of every skill is EXACTLY 5 items:

1. **`__init__.py`** — scaffolder + integrator for whatever this skill curates (executable, not just metadata)
2. **`SKILL.md`** — frontmatter + body
3. **`scripts/`** — executable code directory
4. **`references/`** — docs directory (may contain nested `lessons/`; presence of lessons does NOT change tier)
5. **`<skill_name>.skill`** — packaged zip archive (regenerated by packager; ignored on input, replaced on output)

**NOTHING ELSE at the root.** No `assets/`, no `README.md`, no `CHANGELOG.md`, no `TODO.md`, no `notes.md`, no `INSTALLATION_GUIDE.md`, no OS metadata.

**Why:** during CL-040 I wrote skill-creator's SKILL.md with an "anatomy" diagram that included `assets/` as a valid root item (imported from an outdated 1.0.0 template). User was already-strict on this and had to correct me. Same pass, I imported an `assets` skill from remote — user pointed out `assets` should not be a top-level skill at all. Also 3 existing skills had rogue `assets/` subdirs from the old scaffolding template.

**How to apply:**
- **NO `assets/` ANYWHERE** — not at root, not nested under `references/`, not nested under `scripts/`. Zero exceptions. An "asset" as a category is banned because it duplicates the purpose of scripts/ (executable) and references/ (docs). If a file is a self-contained resource used BY the skill, embed it in the specific script/reference that consumes it, with a purpose-scoped filename.
- **`references/` may ONLY contain** files + `templates/` + `lessons/` subdirs — NO other subdirectories allowed. Enforced by validator.
- The scaffolder (`skill-creator/__init__.py`'s `scaffold()`) MUST NOT mkdir `assets/`
- The validator MUST FAIL any skill with an `assets/` directory found ANYWHERE in the tree
- The validator MUST FAIL if `references/` contains any subdirectory other than `templates/` or `lessons/`
- The auto-fixer (`--fix`) MUST delete `assets/` dirs (they're structurally wrong)
- `references/lessons/` (optional nested) — real usage-captured lessons; empty lessons dir does NOT block enterprise
- `references/templates/` (optional nested) — read-only 0444 skeleton files per validator
- Any file at the root that isn't in the 5-item list gets MOVED to `scripts/` or `references/` by extension (auto-fix) — never renamed in batch
