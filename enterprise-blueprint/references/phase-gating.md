# Phase Gating Reference

Phase gating is the mechanism that stops a multi-phase implementation from
advancing on a broken foundation. Every phase carries three coupled controls: a
**rollback tag**, one or more **feature flags**, and a **validation gate**. A
phase is only "done" when its gate tier passes.

## The Three Controls

### 1. Rollback Tag

Format: `[PHASE-N-v1]`. Anchors the phase to a revertible checkpoint. Every
section that a phase mutates should also carry its own section tag
(`[SECTION-NAME-v1]`, `[MOD-NNN-v1]`) so a rollback can be surgical rather than
wholesale. Tags are updated in the **same commit** as the change they cover —
never after.

### 2. Feature Flags

Format: `FEAT_UPPER_SNAKE_CASE`. A phase's work lands behind a flag that defaults
**off**. This decouples "code merged" from "behaviour active": a phase can be
integrated, tested, and then enabled independently, and disabling the flag is the
first step of any rollback.

Rules:
- Declare the flag in the Module Registry **before** implementing against it.
- No phase advances while its flag is enabled but its gate is failing.
- Rollback procedure always begins `Disable FEAT_* → revert <script> → …`.

### 3. Validation Gate

A concrete, runnable check that must pass before the next phase begins. In this
skill the gate is a `test-runner.py` tier (usually `unit` or `integration`) plus
any phase-specific health command. The gate is binary: PASS advances, FAIL
blocks. "Warnings only" is a PASS; any FAIL is a block.

## Gate Enforcement in Practice

```bash
# Gate for PHASE-N before advancing to PHASE-(N+1)
python3 scripts/test-runner.py integration --json
# exit 0 → record gate PASS in checklist, flip FEAT_* on
# exit 1 → phase stays open; fix, do not advance
```

The checklist generated by `generate_checklist.py` renders each phase's gate as
its final, un-checkable-until-green item, so an operator physically cannot tick a
phase complete without a green gate.

## Rollback Procedure Anatomy

Every phase documents a rollback in this shape:

```
Disable FEAT_<flags> → revert <scripts touched> → <recovery/health command>
```

Because flags default off and tags are committed alongside changes, a rollback is
always: turn the flag off (behaviour reverts instantly), then revert to the
tagged checkpoint if the code itself must go. See
[phase-template.md](phase-template.md) for the full per-phase template and a
worked PHASE-7 example.
