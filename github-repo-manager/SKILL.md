# GitHub Repository Manager

Enterprise-grade GitHub repository management system for bulk operations across all user repositories. Automatically adds licenses, updates repository descriptions/about sections, and enforces repository standards.

## When to Use

- Adding missing licenses to repositories
- Updating repository descriptions/about sections
- Enforcing repository standards (topics, visibility, features)
- Bulk management of user/organization repositories
- Syncing repository metadata from templates
- Auditing repository compliance

## Features

- **License Management**: Detects missing licenses, adds appropriate license files (MIT, Apache-2.0, GPL-3.0, etc.)
- **Description/About Sync**: Updates repository description, website, and topics from templates or inferred from code
- **Bulk Operations**: Process all repositories for a user or organization
- **Dry-run Mode**: Preview changes before applying
- **Template System**: Configurable templates for different project types
- **Compliance Reporting**: Generate reports on repository standards compliance
- **Rate Limit Handling**: Respects GitHub API rate limits with exponential backoff

## Quick Start

```bash
# Dry-run: preview changes across all repos
python3 scripts/github-repo-manager.py --user drdeeks --dry-run

# Apply license to all repos missing one
python3 scripts/github-repo-manager.py --user drdeeks --add-license MIT

# Update descriptions for all repos
python3 scripts/github-repo-manager.py --user drdeeks --update-descriptions

# Full compliance enforcement
python3 scripts/github-repo-manager.py --user drdeeks --enforce-all --config config.yaml
```

## Configuration

Create a `config.yaml` for project-type templates:

```yaml
license: MIT
default_topics:
  - automation
  - github
  - repository-management
project_templates:
  python:
    license: MIT
    topics: ["python", "backend", "api"]
    description_template: "{name} - Python {type} service"
  node:
    license: MIT
    topics: ["nodejs", "javascript", "frontend"]
    description_template: "{name} - Node.js {type} application"
  rust:
    license: MIT
    topics: ["rust", "systems", "cli"]
    description_template: "{name} - Rust {type} tool"
  go:
    license: MIT
    topics: ["go", "golang", "backend"]
    description_template: "{name} - Go {type} service"
```

- Processes in batches of 10 repos
- Respects `Retry-After` headers
