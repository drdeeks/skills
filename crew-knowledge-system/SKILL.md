# Crew Knowledge System

**Two crew modes, one knowledge fabric.**

| Mode | Workspace | Persistence | Secrets | Use Case |
|------|-----------|-------------|---------|----------|
| **Development Crew** | Shared `crew/` root, single workspace | Ephemeral (session) | Placeholders only | Spin up → communicate → negotiate → figure out → tear down |
| **Production Crew** | `crew/agents/<agent-id>/` per agent | Long-term (git/Ventoy) | Real secrets via enforcer | Persistent agents, long-term engagement, cross-crew communication |

## Document Format Standard

Every crew document follows this format:

```markdown
---
# REQUIRED METADATA (auto-injected by tools)
agent_id: "ui-a1b2"
agent_type: "ui"
crew_id: "hackathon-2026"
crew_mode: "development"  # or "production"
doc_type: "decision"      # decision | spec | learning | communication | reasoning | sync
category: "architecture"  # architecture | api | ui | infra | process | debugging | research
tags: ["authentication", "oauth", "session-management"]
timestamp: "2026-07-05T14:32:17Z"
session_id: "sess-7f9a2b1c"
parent_doc: ""          # for thread/reply chains
reply_to: ""            # for communication replies
---

# Document Title

## Context / Problem Statement
[Why this document exists, what triggered it]

## Reasoning / Thinking Process
[Agent's internal reasoning - REQUIRED for decisions/specs/learnings]
- Considered options: [A, B, C]
- Tradeoffs evaluated: [X vs Y]
- Why chosen: [rationale]
- Risks identified: [mitigations]

## Decision / Specification / Learning
[The actual content]

## Implications / Follow-up
[What this affects, what needs to happen next]

## Related
- [[wikilink-to-related-doc]]
- [[another-entity]]
```

**Auto-appended on every write:**
```markdown
---
*Generated by agent `ui-a1b2` (ui) at 2026-07-05T14:32:17Z in crew `hackathon-2026` (development)*
*Session: sess-7f9a2b1c | Category: architecture | Doc ID: dec-auth-oauth-001*
```

## Knowledge Indexer

**Scans crew root, categorizes, builds search indexes.**

```bash
# Index entire crew (dev or prod)
bash scripts/crew-indexer.sh index

# Index specific agent's knowledge
bash scripts/crew-indexer.sh index-agent ui-a1b2

# Search crew knowledge
bash scripts/crew-indexer.sh search "oauth authentication"
bash scripts/crew-indexer.sh search --category architecture "session management"
bash scripts/crew-indexer.sh search --agent ui-a1b2 "design system"
```

**Category Schema:**
| Category | Sub-categories | Doc Types |
|----------|----------------|-----------|
| `architecture` | system-design, data-model, api-design, infra | decisions, specs |
| `api` | rest, graphql, grpc, webhooks, auth | specs, decisions |
| `ui` | components, design-system, accessibility, state | specs, learnings |
| `infra` | deployment, monitoring, scaling, ci-cd | decisions, specs |
| `process` | workflow, communication, review, planning | learnings, communications |
| `debugging` | root-cause, performance, errors, testing | reasoning, learnings |
| `research` | exploration, spikes, evaluation, comparison | learnings, specs |

**Index Output:**
```
crew/index/
├── by-category/
│   ├── architecture.json      # {doc_id, agent_id, timestamp, title, tags}
│   ├── api.json
│   └── ...
├── by-agent/
│   ├── ui-a1b2.json
│   └── ...
├── by-tag/
│   ├── authentication.json
│   └── ...
├── semantic/                  # vector embeddings (if enabled)
│   ├── vectors.faiss
│   └── metadata.json
└── crew-manifest.json         # master index
```

## Agent Communication Protocol

**Structured, traceable, searchable.**

```bash
# Send message to another agent
bash scripts/crew-comm.sh send \
  --from ui-a1b2 \
  --to integration-b2c3 \
  --type "sync" \
  --subject "API contract for user auth" \
  --body "Need to align on OAuth2 token refresh endpoint..."

# Reply to thread
bash scripts/crew-comm.sh reply \
  --thread comm-7f9a2b1c \
  --from integration-b2c3 \
  --body "Proposing /auth/refresh with 15min expiry..."

# Broadcast to all agents
bash scripts/crew-comm.sh broadcast \
  --from lead-x1y2 \
  --subject "Phase gate: planning → confirmation" \
  --body "All agents: confirm readiness for confirmation phase"

# View thread
bash scripts/crew-comm.sh thread comm-7f9a2b1c
```

**Message Format (stored in `crew/shared/communications/messages/`):**
```markdown
---
message_id: "comm-7f9a2b1c"
thread_id: "thread-3e8d1a9f"
from_agent: "ui-a1b2"
from_type: "ui"
to_agent: "integration-b2c3"
to_type: "integration"
crew_id: "hackathon-2026"
type: "sync"          # sync | question | proposal | decision | broadcast | alert
subject: "API contract for user auth"
priority: "normal"    # low | normal | high | urgent
timestamp: "2026-07-05T14:35:22Z"
tags: ["api", "oauth", "contract"]
---

## Message

Need to align on OAuth2 token refresh endpoint. Current spec shows `/auth/refresh` 
but we need to confirm:
- Expiry window (proposing 15min)
- Refresh token rotation policy
- Error response format

## Context
- Related decision: [[dec-auth-oauth-001]]
- Blocking: UI integration tests
- Deadline: confirmation phase gate
```

## Semantic Search Integration

**Optional Turbopuffer integration for fuzzy recall:**

```bash
# Enable semantic vectors (requires TURBOPUFFER_API_KEY, OPENAI_API_KEY)
export CREW_SEMANTIC_ENABLED=true
bash scripts/crew-indexer.sh rebuild --semantic

# Semantic search
bash scripts/crew-indexer.sh semantic-search "how do we handle token expiry"

# Hybrid (keyword + semantic)
bash scripts/crew-indexer.sh hybrid-search "oauth refresh token"
```

**Vector Schema:**
- Chunk size: 3500 chars, 350 char overlap
- Embedding: text-embedding-3-large (3072 dim)
- Metadata: agent_id, doc_type, category, tags, timestamp
- Parent tracking for chunk reassembly

## Pitfalls & Lessons Learned

### ❌ Dev crew agents created in global $HOME/agents but transition expects them in crew workspace
The `init-dev-crew.sh` spawns agents in `$HOME/agents/<agent-id>/` (global enforcer workspace) but the dev crew workspace at `$CREW_DIR` only has shared directories. The `crew.json` MUST be updated with actual agent workspace paths after spawning. The transition script reads `crew.json.agents[]` to find source workspaces — if missing, transition fails silently (empty agent dirs in prod).

**Fix:** `init-dev-crew.sh` now scans `$HOME/agents/` after spawning and updates `crew.json` with `{"agent_id": "...", "workspace": "$HOME/agents/...", "crew_id": "..."}` entries.

### ❌ Production crew uses per-agent subdirs, dev crew uses shared — transition must restructure
Dev: `$CREW_DIR/shared/knowledge/`, `$CREW_DIR/shared/communications/`
Prod: `$CREW_DIR/agents/<agent-id>/knowledge/`, `$CREW_DIR/agents/<agent-id>/.agent/`

The `transition-crew.py` handles this by:
1. Reading agent workspaces from source `crew.json`
2. Copying each agent's `.agent/`, `memory/`, `tools/` to target `agents/<id>/`
3. Copying shared knowledge/comms to target `shared/`
4. Updating `agent.json` with new paths and crew_id

### ❌ Knowledge indexer needs crew.json to detect crew root
`crew-indexer.sh` auto-detects crew workspace by walking up from cwd looking for `crew.json`. If you `cd` into an agent subdirectory, it won't find the crew root. Always run from crew root or set `CREW_WORKSPACE` env var.

### ❌ Document format requires agent_id auto-detection from workspace
`crew-doc.sh` auto-detects agent_id by checking which agent workspace contains the current directory. In dev mode (shared workspace), this fails — must pass `--agent-id` explicitly. In prod mode (per-agent subdirs), it works automatically.

### ❌ Semantic search requires TURBOPUFFER_API_KEY + OPENAI_API_KEY
The `CREW_SEMANTIC_ENABLED=true` flag alone is insufficient. Both API keys must be set. The indexer gracefully degrades to keyword-only search if keys missing, but `rebuild --semantic` will fail with cryptic error if keys not configured.

### ❌ Communication protocol stores messages in shared/ for dev, but agent/ for prod
Dev: `$CREW_DIR/shared/communications/messages/`
Prod: `$CREW_DIR/agents/<agent-id>/communications/messages/` (per-agent) + `$CREW_DIR/shared/communications/` (broadcasts)

The `crew-comm.sh` handles this by checking crew mode from `crew.json` and routing accordingly. When transitioning dev→prod, messages are copied to both locations.

### ❌ Crew sync must handle dev vs prod directory structures differently
`crew-sync.sh --full` in dev mode syncs `shared/knowledge/` → `shared/index/` and each agent's `knowledge/` → `shared/knowledge/`. In prod mode, it syncs each `agents/<id>/knowledge/` → `shared/knowledge/` and `shared/communications/` → each `agents/<id>/communications/`. The `--reindex` flag rebuilds the appropriate index structure.

### ❌ All 6 hackathon projects verified as completely independent
Each project (mnemosyne, autopilot, aires, agora, edgewalker, edgewalker, federation) has zero cross-project imports. The `shared/` directory contains only federation-level utilities consumed by the federation server. `grep -r "shared/" */src/` returns empty for all projects. Root `package.json` provides `demo:all` runner for CI/CD.
