# MLOps Tools Reference

## Training Frameworks
- **PyTorch Lightning**: Structured training, distributed
- **Hugging Face Transformers**: Pre-trained models, fine-tuning
- **TensorFlow/Keras**: Production-ready pipelines
- **JAX/Flax**: High-performance research

## Experiment Tracking
- **MLflow**: Open source, local/remote tracking
- **Weights & Biases**: Cloud-based, collaboration
- **ClearML**: Open source, auto-tracking
- **Neptune**: Metadata management

## Model Registry
- **MLflow Model Registry**: Versioning, staging
- **Hugging Face Hub**: Model sharing, versioning
- **Weights & Biases Registry**: Integrated tracking
- **Custom**: S3/GCS + metadata DB

## Deployment
- **TensorFlow Serving**: High-performance TF serving
- **TorchServe**: PyTorch model serving
- **Triton Inference Server**: Multi-framework
- **BentoML**: Unified model serving
- **vLLM**: High-throughput LLM serving

## Monitoring
- **Prometheus + Grafana**: Metrics, dashboards
- **Evidently AI**: Data drift, model monitoring
- **WhyLabs**: ML observability platform
- **Custom**: Logging + alerting

## Orchestration
- **Kubeflow**: Kubernetes-native ML pipelines
- **Airflow**: General workflow orchestration
- **Prefect**: Modern workflow orchestration
- **Dagster**: Data orchestration

## Feature Stores
- **Feast**: Open source feature store
- **Hopsworks**: Enterprise feature platform
- **Tecton**: Managed feature store
- **Custom**: Redis/DB + schema
