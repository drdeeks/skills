# Crew Document Format Specification

## Overview
Every crew document follows a standardized format with required metadata for traceability, searchability, and semantic indexing.

## Frontmatter (Required)

```yaml
---
# Agent identification (auto-injected)
agent_id: "ui-a1b2"
agent_type: "ui"
crew_id: "hackathon-2026"
crew_mode: "development"  # or "production"

# Document classification
doc_type: "decision"      # decision | spec | learning | communication | reasoning | sync
category: "architecture"  # architecture | api | ui | infra | process | debugging | research
tags: ["oauth", "authentication", "session-management"]

# Timestamps
timestamp: "2026-07-05T14:32:17Z"
session_id: "sess-7f9a2b1c"

# Threading (for communications)
parent_doc: ""            # for thread/reply chains
reply_to: ""              # for communication replies

# Optional
priority: "normal"        # low | normal | high | urgent (for communications)
related: []               # array of related doc IDs
---
```

## Document Types

### Decision Document
```markdown
# Document Title

## Context / Problem Statement
[Why this decision exists, what triggered it]

## Reasoning / Thinking Process
[REQUIRED - Agent's internal reasoning]
- Considered options: [A, B, C]
- Tradeoffs evaluated: [X vs Y]
- Why chosen: [rationale]
- Risks identified: [mitigations]

## Decision
[The actual decision made]

## Implications / Follow-up
[What this affects, what needs to happen next]

## Related
- [[wikilink-to-related-doc]]
```

### Spec Document
```markdown
# Document Title

## Overview
[What this spec covers]

## Requirements
- Functional: [...]
- Non-functional: [...]

## Design / Architecture
[Technical design details]

## API / Interface
[If applicable]

## Implementation Notes
[Guidance for implementers]

## Testing Criteria
[How to validate]
```

### Learning Document
```markdown
# Document Title

## What Happened
[Event, bug, discovery, experiment]

## Root Cause / Analysis
[Why it happened]

## Lesson Learned
[The durable insight]

## Prevention / Application
[How to apply this going forward]

## Related
-related
- [[wikilink]]
```

### Communication Document
```markdown
# Document Title

## Message
[The communication content]

## Context
- Thread: [[thread-id]]
- Related: [[doc-id]]
```

### Reasoning Document
```markdown
# Document Title

## Problem / Question
[What is being reasoned about]

## Options Considered
1. Option A - [pros/cons]
2. Option B - [pros/cons]
3. Option C - [pros/cons]

## Analysis
[Deep dive reasoning]

## Recommendation
[Recommended path with rationale]
```

### Sync Document
```markdown
# Document Title

## Status Update
[Current state]

## Completed
- [x] Item 1
- [x] Item 2

## In Progress
- [ ] Item 3

## Blockers
- Blocker description

## Next Steps
- [ ] Next action
```

## Auto-Appended Footer

Every document gets this appended on write:

```markdown
---
*Generated by agent `ui-a1b2` (ui) at 2026-07-05T14:32:17Z in crew `hackathon-2026` (development)*
*Session: sess-7f9a2b1c | Category: architecture | Doc ID: dec-auth-oauth-001*
```

## File Naming Convention

```
{doc_type}-{category}-{slug}-{short-id}.md
```

Examples:
- `dec-architecture-auth-oauth-001.md`
- `spec-api-user-profile-v2-042.md`
- `learn-debugging-race-condition-session-017.md`
- `comm-sync-confirmation-phase-003.md`

## Category Taxonomy

| Category | Sub-categories | Typical Doc Types |
|----------|----------------|-------------------|
| architecture | system-design, data-model, api-design, infra | decisions, specs |
| api | rest, graphql, grpc, webhooks, auth | specs, decisions |
| ui | components, design-system, accessibility, state | specs, learnings |
| infra | deployment, monitoring, scaling, ci-cd | decisions, specs |
| process | workflow, communication, review, planning | learnings, comms |
| debugging | root-cause, performance, errors, testing | reasoning, learnings |
| research | exploration, spikes, evaluation, comparison | learnings, specs |

## Tagging Guidelines

- Use lowercase, hyphen-separated
- Max 5 tags per document
- Include at least one category-aligned tag
- Use consistent vocabulary across crew
