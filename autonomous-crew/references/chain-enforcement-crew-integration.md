# Chain Enforcement Integration with Crew Manager

## Overview
The crew-manager.py creates crew-specific chains in `.crew-<name>/.blueprint-chain/` that must work with the existing chain_enforce.py system.

## Chain Discovery Priority

chain_enforce.py searches for chains in this order:
1. `.crew-*/.blueprint-chain/` (crew-manager chains - highest priority)
2. `.blueprint-chain/` (legacy blueprint chains)
3. `.chain/` (loop-enforcer native chains)

## Chain Format Differences

### crew-manager chain format (mnemosyne-blueprint.json)
```json
{
  "name": "mnemosyne-blueprint",
  "project": "$HOME/qwen-cloud-2026/mnemosyne",
  "crew": "$HOME/qwen-cloud-2026/mnemosyne/.crew-mnemosyne-crew",
  "created_at": "2026-07-09T05:05:47Z",
  "blueprint": "$HOME/qwen-cloud-2026/mnemosyne/blueprint.md",
  "steps": [
    {
      "index": 0,
      "path": "$HOME/qwen-cloud-2026/mnemosyne/.crew-mnemosyne-crew/.blueprint-chain/phase-00-Foundation-and-Infrastructure.marker",
      "state": "active",
      "validator": null
    }
  ]
}
```

### loop-enforcer chain format (.chain/mnemosyne.json)
```json
{
  "name": "mnemosyne",
  "steps": [
    {"path": ".../phase-00.marker", "state": "complete"},
    {"path": ".../phase-01.marker", "state": "active"}
  ]
}
```

## chain_enforce.py Adaptations

### find_chain() - Modified to search crew directories first
```python
def find_chain(project_dir):
    search_dirs = []
    
    # 1. Crew-manager chains (highest priority)
    for item in os.listdir(project_dir):
        if item.startswith(".crew-") and os.path.isdir(os.path.join(project_dir, item)):
            search_dirs.append(os.path.join(item, ".blueprint-chain"))
    
    # 2. Standard locations
    search_dirs.extend([".blueprint-chain", ".chain"])
    
    for chain_dir_name in search_dirs:
        chain_dir = os.path.join(project_dir, chain_dir_name)
        if not os.path.isdir(chain_dir):
            continue
        for f in os.listdir(chain_dir):
            if f.endswith(".json") and "blueprint" in f:
                # ... load and return
```

### cmd_check() - Handles both formats
```python
if ".crew-" in chain_path:
    # Crew-manager format: check state directly from JSON
    output = f"Marker exists: {marker_exists}"
    rc = 0
else:
    # Loop-enforcer format: run chain.py check
    output, rc = run_chain_cmd("check", project_dir, chain_name, marker_path)
```

### cmd_complete() - Updates crew-manager chain state
```python
if ".crew-" in chain_path:
    # Update state in chain JSON directly
    step["state"] = "complete"
    if idx + 1 < len(chain_data["steps"]):
        chain_data["steps"][idx + 1]["state"] = "active"
    with open(chain_path, "w") as f:
        json.dump(chain_data, f, indent=2)
else:
    # Use chain.py complete
    run_chain_cmd("complete", project_dir, chain_name, marker_path)
```

## Phase Validator Integration

Each phase has a validator script generated by generate-phase-validators.py:
```
.crew-mnemosyne-crew/validators/
├── phase-00-validator.py
├── phase-01-validator.py
...
```

Validators check deliverables from checklist.md before allowing phase completion.

## Kanban Wiring

wire-kanban-to-chain.py creates mapping in `.crew-<name>/config/kanban-wiring.json`:
```json
{
  "crew_dir": "/project/.crew-mnemosyne-crew",
  "project_dir": "/project",
  "chain_dir": "/project/.crew-mnemosyne-crew/.blueprint-chain",
  "kanban_db": "$HOME/.hermes/kanban.db",
  "created_at": "2026-07-09T05:05:47Z"
}
```

This enables kanban tasks to be linked to chain phases.

## Pitfalls to Avoid

❌ Don't assume chain is in `.chain/` only
❌ Don't hardcode phase marker paths
❌ Don't skip validator execution on complete
✅ Use find_chain() which handles all formats
✅ Run phase validator before marking complete
✅ Update kanban task status when chain advances
